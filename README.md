# Catalog Crawler

Catalog Crawler discovers product/catalog URLs from merchant sitemaps and scrapes Product JSON-LD structured data from those pages.

The project is intentionally small: sitemap discovery and page scraping share a browser-fingerprinted HTTP client, and crawl state is stored in per-domain SQLite databases.

## What It Does

- Reads `robots.txt` and discovers declared sitemap URLs.
- Resolves nested sitemap indexes into final page URLs.
- Stores sitemap records, discovered pages, scrape status, errors, and timing in SQLite.
- Scrapes `application/ld+json` blocks and stores raw Schema.org `Product` objects.
- Supports resume behavior by skipping pages already marked visited.

## Setup

```bash
go mod tidy
```

The project uses `sqlc` generated code. Regenerate DB code after changing SQL files:

```bash
sqlc generate
```

## Commands

Show available commands:

```bash
go run ./cmd/catalog-crawler --help
```

### Resolve Sitemaps

Resolve sitemap URLs from a site's `robots.txt` and persist results to SQLite:

```bash
go run ./cmd/catalog-crawler resolve-sitemaps \
  --robots-url https://www.biba.in/robots.txt
```

Default database path:

```text
data/<host>/scraper.db
```

Useful flags:

```bash
--db data/www.biba.in/scraper.db
--output data
--concurrency 4
--min-delay-ms 0
--max-delay-ms 0
--http-mode tls
--tls-profile chrome_146
--http-timeout 30s
```

### Scrape Structured Data

Scrape pending pages from SQLite:

```bash
go run ./cmd/catalog-crawler scrape-structured-data \
  --db data/www.biba.in/scraper.db \
  --limit 10
```

Scrape specific URLs:

```bash
go run ./cmd/catalog-crawler scrape-structured-data \
  --url https://www.biba.in/example-product.html
```

Seed from an older `pages.json` export, then scrape:

```bash
go run ./cmd/catalog-crawler scrape-structured-data \
  --input data/www.biba.in/pages.json \
  --limit 10
```

For larger runs, keep concurrency modest and use delay/jitter:

```bash
go run ./cmd/catalog-crawler scrape-structured-data \
  --db data/www.biba.in/scraper.db \
  --limit 0 \
  --parallelism 2 \
  --delay-ms 500 \
  --random-delay-ms 500
```

Use `--debug` only for small troubleshooting runs. It prints per-page status lines:

```text
[scraped]  [1201/5634]  200  531ms  https://example.com/product
[failed]   [1202/5634]  403  212ms  https://example.com/product  unexpected status 403 Forbidden
[skipped]  [1203/5634]  0    0ms    https://example.com/product  URL outside allowed domains
```

### HTTP Client Options

All outbound requests use `bogdanfinn/tls-client` by default:

```bash
--http-mode tls
```

To opt out and use Go's standard HTTP client:

```bash
--http-mode stdlib
```

TLS mode defaults to:

```bash
--tls-profile chrome_146
--http-timeout 30s
```

Additional troubleshooting flags:

```bash
--tls-debug
--force-http1
--disable-http3
--no-redirects
```

## SQLite Data

Each domain gets its own database:

```text
data/www.biba.in/scraper.db
data/www.zara.com/scraper.db
```

Main tables:

- `sitemaps`: sitemap URL, parent source, status, type, error, delay, duration.
- `pages`: discovered URL, source sitemap, `lastmod`, scrape status, HTTP status, duration, error, visited timestamp.
- `structured_products`: raw Product JSON-LD linked to a page.

Resume behavior is based on `pages.visited_at`. Pages with `visited_at IS NOT NULL` are skipped by default.

## Development

Run tests:

```bash
go test ./...
```

After editing `internal/db/schema.sql` or `internal/db/queries.sql`:

```bash
sqlc generate
go test ./...
```

## Notes

- JSON exports are no longer generated by default; SQLite is the source of truth.
- Crawl state is managed by the application database.
- Large sites can expose millions of sitemap URLs. Use filters or small limits before running full scrapes.
